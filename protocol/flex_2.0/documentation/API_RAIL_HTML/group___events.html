RAIL_EVENT_RSSI_AVERAGE_DONE 

Occurs when the AGC averaged RSSI is done.

It occurs in response to RAIL_StartAverageRssi() to indicate that the hardware has completed averaging. Call RAIL_GetAverageRssi to get the result.

RAIL_EVENT_RX_ACK_TIMEOUT 

Notifies the application when searching for an ACK has timed out.

This event occurs whenever the timeout for searching for an ACK is exceeded.

RAIL_EVENT_RX_FIFO_ALMOST_FULL 

Occurs when the receive FIFO exceeds the configured threshold value.

Call RAIL_GetRxFifoBytesAvailable to get the number of bytes available.

RAIL_EVENT_RX_PACKET_RECEIVED 

Occurs whenever a packet is received.

Call RAIL_GetRxPacketInfo() to get basic packet information along with a handle to this packet for subsequent use with RAIL_PeekRxPacket(), RAIL_GetRxPacketDetails(), RAIL_HoldRxPacket(), and RAIL_ReleaseRxPacket() as needed.

If RAIL_RX_OPTION_IGNORE_CRC_ERRORS is set, this event also occurs for packets with CRC errors.

RAIL_EVENT_RX_PREAMBLE_DETECT 

Event for preamble detection.

RAIL_EVENT_RX_SYNC1_DETECT 

Event for detection of the first sync word.

RAIL_EVENT_RX_SYNC2_DETECT 

Event for detection of the second sync word.

RAIL_EVENT_RX_FRAME_ERROR 

Event for detection of frame errors.

For efr32xg1x parts, frame errors include violations of variable length minimum/maximum limits, frame coding errors, and CRC errors. If RAIL_RX_OPTION_IGNORE_CRC_ERRORS is set, RAIL_EVENT_RX_FRAME_ERROR will not occur for CRC errors.

RAIL_EVENT_RX_FIFO_OVERFLOW 

Occurs when RX buffer is full.

RAIL_EVENT_RX_ADDRESS_FILTERED 

Occurs when a packet is address filtered.

RAIL_EVENT_RX_TIMEOUT 

Occurs when an RX event times out.

RAIL_EVENT_RX_SCHEDULED_RX_END 

Occurs when the scheduled RX window ends.

RAIL_EVENT_RX_PACKET_ABORTED 

An event for an aborted packet.

It is triggered when a more specific reason isn't known for why the packet was aborted (e.g. RAIL_EVENT_RX_ADDRESS_FILTERED).

RAIL_EVENT_RX_FILTER_PASSED 

Occurs when the packet has passed any configured address and frame filtering options.

RAIL_EVENT_RX_TIMING_LOST 

Occurs when modem timing is lost.

RAIL_EVENT_RX_TIMING_DETECT 

Occurs when modem timing is detected.

RAIL_EVENT_IEEE802154_DATA_REQUEST_COMMAND 

Indicates a Data Request is being received when using IEEE 802.15.4 functionality.

This occurs when the command byte of an incoming frame is for a data request, which requests an ACK. This callback is called before the packet is fully received to allow the node to have more time to decide whether to set the frame pending in the outgoing ACK. This event only ever occurs if the RAIL IEEE 802.15.4 functionality is enabled.

Call RAIL_IEEE802154_GetAddress to get the source address of the packet.

RAIL_EVENT_TX_FIFO_ALMOST_EMPTY 

Occurs when the transmit FIFO falls under the configured threshold value.

It only occurs if a rising edge occurs across this threshold. This event does not occur on initailization or after resetting the transmit FIFO with RAIL_ResetFifo(). Call RAIL_GetTxFifoSpaceAvailable to get the number of bytes available in the transmit FIFO at the time of the callback dispatch.

RAIL_EVENT_TX_PACKET_SENT 

Interrupt level event to signify when the packet was sent.

Call RAIL_GetTxPacketDetails() to get information about the packet that was transmitted.

Note
that this structure is only valid during the timeframe of the RAIL_Config_t::eventsCallback.
RAIL_EVENT_TXACK_PACKET_SENT 

An interrupt level event to signify when the packet was sent.

Call RAIL_GetTxPacketDetails() to get information about the packet that was transmitted.

Note
This structure is only valid during the timeframe of the RAIL_Config_t::eventsCallback.
RAIL_EVENT_TX_ABORTED 

Occurs when a TX is aborted by the user.

RAIL_EVENT_TXACK_ABORTED 

Occurs when a TX ACK is aborted by the user.

RAIL_EVENT_TX_BLOCKED 

Occurs when a TX is blocked by something like PTA or RHO.

RAIL_EVENT_TXACK_BLOCKED 

Occurs when a TX ACK is blocked by something like PTA or RHO.

RAIL_EVENT_TX_UNDERFLOW 

Occurs when the TX buffer underflows.

RAIL_EVENT_TXACK_UNDERFLOW 

Occurs when the buffer used for TX acking underflows.

RAIL_EVENT_TX_CHANNEL_CLEAR 

Occurs when CCA/CSMA/LBT succeeds.

RAIL_EVENT_TX_CHANNEL_BUSY 

Occurs when CCA/CSMA/LBT fails.

RAIL_EVENT_TX_CCA_RETRY 

Occurs when a CCA check is being retried.

RAIL_EVENT_TX_START_CCA 

Occurs when a clear channel assessment (CCA) is begun.

RAIL_EVENT_CONFIG_UNSCHEDULED 

Event for when the scheduler switches away from this configuration.

RAIL_EVENT_CONFIG_SCHEDULED 

Event for when the scheduler switches to this configuration.

RAIL_EVENT_SCHEDULER_STATUS 

Event for when the status of the scheduler changes.

RAIL_EVENT_CAL_NEEDED 

Notifies the application that a calibration is needed.

It occurs whenever the RAIL library detects that a calibration is needed. An application determines a valid window to call RAIL_Calibrate().

RAIL_EVENTS_ALL 

Value representing all possible events.

RAIL_Events_t  mask,
RAIL_Events_t  events 
)

Configures radio events.

Parameters
[in]railHandleA RAIL instance handle.
[in]maskBitmask containing which events should be modified.
[in]eventsDefine which events should trigger RAIL_Config_t::eventsCallback The full list of available callbacks can be found by looking at the RAIL_EVENT_* set of defines.
Returns
Status code indicating success of the function call.

Sets up which radio interrupts generate a RAIL event. The full list of options is in RAIL_Events_t.
